h1. Wie funktioniert Bloonix

h2. Übersicht über alle Komponenten

p. Hier sehen Sie eine grobe Übersicht über alle Komponenten. Da der Bloonix-Server und die Bloonix-WebGUI serverseitig als externe FastCGI-Applikation laufen, sind diese hochskalierbar und können auf mehreren Servern installiert werden. Alles, was Sie dann noch benötigen, ist ein HTTPS-Offloader und Loadbalancer, um die Anfragen zu verteilen.

h1. Wie funktioniert das nun mit der Überwachung von Hosts und Services

h2. Einrichtung von Hosts und Services im Webinterface

p. Im Webinterface werden alle Hosts und Services eingerichtet, die Sie überwachen möchten. Ein Host kann im Kontext von Bloonix ein Server, ein virtueller Server, ein Router, ein Switch oder auch ein Drucker sein. Jeder Host besitzt einen Hostnamen und eine IP-Adresse. Wenn ein Host im Webinterface eingerichtet wird, erhält dieser eine eindeutige ID und ein dazugehöriges Passwort. Darüber hinaus werden Service-Checks eingerichtet, zum Beispiel zur Überwachung der CPU-Auslastung, des Speicherverbrauchs oder der Auslastung der Netzwerkinterfaces.

!{width: 640px; margin: 40px 0;}/public/img/bloonix-howto-webgui.png!

h2. Einrichtung des Agenten

p. Ist ein Host im Webinterface eingerichtet, so muss als nächstes der Bloonix-Agent auf dem zu überwachenden Server installiert und konfiguriert werden. Zusätzlich müssen die Plugins zur Überwachung der Services installiert werden. Möchten Sie zum Beispiel eine MySQL-Datenbank überwachen, so installieren Sie das Paket __bloonix-plugins-mysql__.

p. Damit der Agent nun weiß, welche Services dieser überwachen soll, wird die Host-ID und das Passwort des Hosts, welche im Webinterface zu finden sind, in der Konfigurationsdatei des Agenten eingetragen. Zusätzlich benötigt der Agent die Adresse (Hostname oder IP) des Bloonix-Servers.

h2. Ablauf der Überwachung

p. Was der Agent nun macht ist folgendes:

# Der Bloonix-Agent verbindet sich mit dem Bloonix-Server und übersendet die Host-ID und das Passwort.
# Der Bloonix-Server übersendet dem Agenten die Services, die für die Host-ID eingerichtet sind.
# Der Bloonix-Agent überprüft die Services und übersendet den Status und die Statistiken der Services an den Bloonix-Server.
# Der Bloonix-Server validiert die Daten, speichert diese in der Datenbank und informiert gegebenfalls einen Administrator per E-Mail oder SMS.

!{width: 640px; margin: 40px 0;}/public/img/bloonix-howto-agent-server.png!

h2. Überwachung von Windows-Systemen, Routern, Switches, Drucker

p. Da der Bloonix-Agent nicht für Windows-Systeme zur Verfügung steht und auch nicht auf Netzwerkkomponenten installiert werden kann, können diese Systeme nur über SNMP oder externe TCP/UDP Schnittstellen überwacht werden.

p. Hierfür wird der Bloonix-Agent auf einem zentralen Linux-Server installiert, der in Ihrer lokalen Infrastruktur integriert ist, also in dem LAN Ihres Unternehmens. Von dort aus werden alle Netzwerksysteme und Server über SNMP, HTTP und anderen Protokollen überwacht. Der Bloonix-Agent ist so konzipiert, dass er beliebig viele Hosts und Services überwachen kann.
